## 一、系统架构总览

采用 **模块化单体 + AI 编排** 架构（前端 / 网关 / 服务 / AI 引擎 / 数据 五层逻辑分层；后端为单体 FastAPI，按模块拆分 router/service，未拆分独立微服务进程）：

```
┌─────────────────────────────────────────────────────────┐
│                    Frontend Layer                        │
│  Vue3 SPA (消费者端) + Vue3 Admin (管理端)                │
│  共享 Component Library · Pinia Store · WebSocket Client │
└──────────────────────┬──────────────────────────────────┘
                       │ HTTPS/WSS
┌──────────────────────┴──────────────────────────────────┐
│                    Gateway Layer                         │
│  Nginx: 反向代理/路由/限流/CORS · 后端中间件: JWT验证     │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────────────┐
│                   Service Layer (FastAPI)                │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐      │
│  │Product  │ │Order    │ │User     │ │Ticket   │      │
│  │Service  │ │Service  │ │Service  │ │Service  │      │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘      │
│       └───────────┴───────────┴───────────┘            │
│        模块直接调用（单体内）· RabbitMQ 异步事件(二期起) │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────────────┐
│                    AI Engine Layer                       │
│  ┌─────────────────────────────────────────────────┐   │
│  │              Master Agent Router                 │   │
│  │  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐   │   │
│  │  │客服Agent│ │推荐Agent│ │订单Agent│ │营销Agent│   │   │
│  │  └───┬────┘ └───┬────┘ └───┬────┘ └───┬────┘   │   │
│  └──────┼──────────┼──────────┼──────────┼───────┘   │
│         │          │          │          │            │
│  ┌──────┴──────────┴──────────┴──────────┴───────┐   │
│  │  LangGraph · LlamaIndex · RAG · LLM Pool       │   │
│  └────────────────────────────────────────────────┘   │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────┴──────────────────────────────────┐
│                     Data Layer                           │
│  MySQL 8.0  │  ChromaDB  │  Redis 7  │  RabbitMQ        │
└─────────────────────────────────────────────────────────┘
```

## 二、技术选型详情

### 2.1 前端

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|---------|
| Vue 3 | 3.4+ | 响应式框架 | Composition API，TS 支持好 |
| Vite | 5.0+ | 构建工具 | 热更新极快，SSR 友好 |
| Element Plus | 2.4+ | UI 组件库 | 企业级组件丰富，中文社区活跃 |
| Pinia | 2.1+ | 状态管理 | 比 Vuex 轻量，TS 原生支持 |
| Axios | 1.6+ | HTTP 请求 | 拦截器方便，生态成熟 |
| ECharts | 5.5+ | 数据图表 | 数据看板核心依赖 |
| WebSocket | 原生 | 实时通信 | 客服聊天、物流推送 |

### 2.2 后端

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|---------|
| Python | 3.10+ | 编程语言 | AI 生态第一选择 |
| FastAPI | 0.109+ | Web 框架 | 异步原生、自动文档、Pydantic 集成 |
| SQLAlchemy | 2.0+ | ORM | 异步支持、迁移工具 Alembic |
| Pydantic | 2.5+ | 数据验证 | FastAPI 原生配合，类型安全 |
| Celery | 5.3+ | 异步任务 | 推荐计算、批量入库、报表生成 |
| Redis | 7.0+ | 缓存/会话 | 购物车、Session、排行榜、限流 |
| RabbitMQ | 3.12+ | 消息队列 | 服务间解耦、Agent 事件通知 |

### 2.3 AI 引擎

| 技术 | 用途 | 选型理由 |
|------|------|---------|
| LangGraph | Agent 编排 | 有状态图编排，适合多 Agent 协作 |
| LlamaIndex | RAG 检索 | 索引构建、混合检索、重排序 |
| ChromaDB | 向量存储 | 轻量嵌入式，适合中小规模 |
| OpenAI / 通义千问 | LLM 底层 | 按场景选择（客服用千问、推荐用 GPT） |
| BGE-M3 | Embedding | 中文语义理解强，1024维 |
| Scikit-learn | 协同过滤 | 轻量推荐，无需 TensorFlow |
| BGE-Reranker | 重排序 | Cross-Encoder 精排 |

### 2.4 部署运维

| 技术 | 用途 |
|------|------|
| Docker Compose | 本地开发环境 |
| Nginx | 反向代理 + 静态资源 |
| GitHub CI/CD | 自动部署 |
| Prometheus + Grafana | 监控告警 |

## 三、部署架构

```
┌────────────── Nginx (:8080) ──────────────────┐
│  /api/v1/* → backend:8010                      │
│  /ws/*     → backend:8010 (WebSocket)          │
│  /admin/*  → 管理端静态产物 (dist)             │
│  /*        → 消费者端静态产物 (dist)           │
└───────────────────────────────────────────────┘

说明：开发环境用 Vite dev server（consumer :5173 / admin :5174）+ 代理；
生产环境前端构建为静态文件由 Nginx 直接托管，不挂 dev server。

Docker Compose Services（端口对齐 CONFIG.md）:
  - backend             (FastAPI :8010)
  - mysql               (MySQL 8.0 :3308→3306)
  - chromadb            (ChromaDB :8001)
  - redis               (Redis 7 :6379, DB=1)
  - nginx               (Nginx :8080, 托管前端静态产物)
  - celery-worker       (Celery Worker，二期起)
  - celery-beat         (Celery Beat 定时任务，二期起)
  - rabbitmq            (RabbitMQ :5672，二期起)
```

## 四、性能指标

| 指标 | 目标值 | 说明 |
|------|:-----:|------|
| API 响应 (P95) | < 200ms | 不含 AI 调用 |
| AI 对话响应 (P95) | < 3s | 含 RAG 检索 + LLM 生成 |
| 推荐响应 (P95) | < 500ms | 含向量检索 + 排序 |
| 并发用户 | 1000+ | 基准测试值 |
| 可用性 | 99.5% | 月宕机 < 3.6 小时 |
