# 模块详细设计 · 智能电商全链路平台

## M1 商品中心

**来源：** 项目八 SPU/SKU 体系 + 项目一树形分类

| 功能 | 优先级 | 说明 |
|------|:---:|------|
| 分类树管理 | P0 | 三级分类，支持增删改查、排序、图标 |
| SPU 管理 | P0 | 商品主信息 CRUD，图文详情（富文本） |
| SKU 管理 | P0 | 多规格组合（颜色×内存×版本），独立价格/库存/条码 |
| 商品属性 | P1 | 可配置属性模板，支持搜索筛选 |
| 商品向量化 | P0 | 创建/更新时自动生成 BGE-M3 Embedding 写入 ChromaDB |
| 批量导入 | P1 | Excel 导入/导出 |
| 库存同步 | P0 | Redis 缓存库存，定时同步 MySQL |

**融合升级点：** 项目一只有简单 CRUD 和分类，融合后升级为完整 SPU/SKU 体系 + 向量化能力。

## M2 智能推荐引擎

**来源：** 项目八 + 项目一知识库的语义检索能力

| 推荐类型 | 算法 | 冷启动 | 更新频率 |
|------|------|:---:|:---:|
| 协同过滤 | UserCF / ItemCF | 热门兜底 | 每日离线计算 |
| 内容推荐 | 商品向量余弦相似度 | ChromaDB 语义 | 实时 |
| 个性化推荐 | 画像加权 + 行为序列 | 新用户热门 | 实时 |
| 热门推荐 | Redis ZSet 排序 | 天然可用 | 每小时 |
| 关联推荐 | 买了又买 / 看了又看 | 热门兜底 | 每日 |

**推荐召回→排序流程：**
```
用户请求 → 多路召回 → 粗排 → 精排 → 重排 → 返回
              │          │       │       │
         协同过滤+语义  LR模型  BGE-Reranker  规则(去重/打散)
```

**融合升级点：** 用户画像现在包含客服对话中提取的偏好信号；推荐失败的商品可通过客服渠道主动介入。

## M3 智能客服系统（大幅增强）

**来源：** 项目一 RAG 引擎 + 成熟客服智能体的话术/素材/风格/自动化能力

### 知识构建与管理

| 功能 | 优先级 | 说明 |
|------|:---:|------|
| 商品知识自动构建 | P0 | 从商品详情图片 OCR+多模态提取材质/参数/属性，自动入知识库 |
| 聊天记录学习 | P1 | 指定金牌客服记录 → LLM 提炼话术模式 → 过滤无效 → 注入知识库 |
| 素材图学习 | P0 | 自动识别尺码表/材质图/尺寸图，手动上传素材组，构建素材图库 |
| 自定义知识配置 | P1 | 支持手动上传物流/发票等规则，用 Prompt 指令让 Agent 理解（无需 FAQ） |
| 知识库管理 | P1 | 后台 CRUD 知识条目、手动修正、批量导入、自动向量化 |
| 全店知识同步 | P1 | 商品更新自动触发知识库向量重建 |

### 智能回复与话术

| 功能 | 优先级 | 说明 |
|------|:---:|------|
| RAG 知识问答 | P0 | ChromaDB + BGE-M3(BM25+向量) → RRF融合 → BGE-Reranker精排 |
| 意图识别 | P0 | **12 类意图**（含商品对比/尺码推荐/催付/投诉等） |
| 多轮对话 | P0 | Redis 上下文管理，支持澄清、追问、意图切换 |
| 千人千面话术 | P0 | 结合用户画像 + 商品卖点 + 促销政策，生成差异化话术 |
| 话术风格定制 | P1 | 品牌可自定义 Agent 角色，给示例即可模拟风格；场景/角色独立配置 |
| 商品对比回复 | P1 | 多维度总结差异点（材质/尺寸/价格/适用场景）并生成回复 |
| 自动素材图发送 | P0 | 识别用户咨询意图后自动匹配并发送商品素材图 |
| 主动推荐 | P0 | 在回答中附带相关商品/搭配推荐 |

### 客服协同

| 功能 | 优先级 | 说明 |
|------|:---:|------|
| 超级助理 | P0 | 3秒生成参考话术，降低新人上手难度 |
| 复制销冠 | P1 | 学习金牌客服记录 → LLM提炼 → 同类问题复现销冠话术 → 团队共享 |
| 人工辅助模式 | P0 | Agent 生成参考话术，客服点"采纳"才发送 |
| 采纳发送模式 | P1 | 客服采纳后系统自动学习修正，后续同类问题自动优化 |
| 全自动发送模式 | P1 | 3秒自动生成+发送，可配置排除场景（退款/负面/优惠） |
| 转人工 | P1 | 3次兜底无效 → 转人工客服 → 创建工单 |

### 新增依赖

| 能力 | 技术扩展 | 复杂度 |
|------|---------|:---:|
| 商品图 OCR + 特征提取 | 千问VL / GPT-4V API | 低（已有 LLM 调用层） |
| 素材图识别+分类 | ChromaDB 多模态向量 | 低 |
| 聊天记录提炼 | Celery 定时任务 + LLM 摘要 | 低 |
| 钉钉/微信/飞书推送 | Webhook URL 配置 | 低 |

**技术栈完全不变：** Vue3 + FastAPI + MySQL + ChromaDB + Redis + LangGraph + LLM API

## M7 数据运营面板

**来源：** 项目一客服统计 + 项目八用户行为分析

### 客服维度
- 咨询量趋势（日/周/月）
- 问题分类分布
- 平均响应时间 / 首次响应时间
- 满意度评分分布
- 热门问题 TOP 20
- 转人工率

### 电商维度
- GMV / 订单量 / 客单价 趋势
- 商品销量 TOP 50
- 分类销售占比
- 优惠券使用率
- 转化漏斗：浏览→加购→下单→支付（每步转化率）

### 用户维度
- 新注册/活跃/留存
- RFM 四象限分布
- 用户画像标签分布
- 复购率

**融合升级点：** 客服数据 + 电商数据 + 用户数据三合一，可以发现"高咨询量但低转化率的商品"等深层洞察。
