# 模块详细设计

## M1 店铺管理

| 功能 | 优先级 | 说明 |
|------|:---:|------|
| 绑定店铺 | P0 | 选择平台类型（mock/taobao/jd/douyin），Mock 模式一键创建 |
| 解绑店铺 | P0 | 级联删除该店铺所有商品/订单/会话 |
| 店铺列表 | P0 | 展示绑定状态、平台类型、上次同步时间、商品/订单数 |
| 手动同步 | P1 | 触发 PlatformConnector.fetch_products/orders |
| Token 管理 | P1 | Redis 缓存外部平台 access_token，提前5分钟自动刷新 |

## M2 商品库

| 功能 | 优先级 | 说明 |
|------|:---:|------|
| 商品列表 | P0 | 表格分页，支持按店铺/分类/关键词/价格筛选 |
| 语义搜索 | P0 | 自然语言搜索（"适合送礼的电子产品"），调用 ChromaDB 向量检索 |
| 商品详情 | P1 | 展示完整信息（标题/价格/描述/图片/分类路径/向量状态） |
| 手动同步 | P1 | 触发单店铺商品同步 → 入库 → 自动向量化 |
| 自动向量化 | P0 | 商品同步后异步调用 BGE-M3 生成 Embedding 写入 ChromaDB |

**向量化文本拼接：** `title + " " + description[:500] + " " + category_path`

## M4 客服工作台

### 布局
```
┌─────────────────────────────────────────────────────────┐
│  ┌──────────┐ ┌──────────────────┐ ┌──────────────────┐│
│  │ 会话列表  │ │    聊天窗口       │ │   AI 推荐话术     ││
│  │          │ │                  │ │                  ││
│  │ ○ 张三   │ │ 张三: 多大码？    │ │ 建议1: 亲，这款   ││
│  │ ● 李四   │ │ 客服: 亲，M码... │ │ ... [复制][发送]  ││
│  │ ○ 王五   │ │ 张三: 有黑色吗？  │ │                  ││
│  │          │ │                  │ │ 建议2: ...       ││
│  │          │ │ [输入框] [发送]   │ │ [复制][发送]     ││
│  └──────────┘ └──────────────────┘ └──────────────────┘│
└─────────────────────────────────────────────────────────┘
```

| 功能 | 优先级 | 说明 |
|------|:---:|------|
| 会话列表 | P0 | 按店铺筛选、未读红点、按时间倒序 |
| 聊天窗口 | P0 | WebSocket 实时推送新消息，支持历史加载 |
| AI 话术面板 | P0 | 实时根据最新买家消息生成 3 条回复建议 |
| 复制话术 | P0 | 一键复制到剪贴板 |
| 模拟发送 | P1 | 调用 MockPlatformConnector.send_message，记录采纳 |
| 会话分配 | P1 | 客服手动认领或管理员分配 |
| 关闭会话 | P1 | 标记已处理 |

### AI 话术生成流程

```
最新买家消息 → ChromaDB 语义检索
    ├── 商品知识 (title+description) → 产品信息匹配
    └── 历史话术 (type='reply') → 相似问题→对应回复
        │
        ▼
    RRF 融合排序 → Top-3
        │
        ▼
    LLM (千问) 结合上下文生成最终建议
        │
        ▼
    返回 3 条话术 → 前端渲染在右侧面板
```
