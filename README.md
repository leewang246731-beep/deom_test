# 智能电商全链路平台 · AI 智能体（增强版）

> 融合「企业智能客服系统」+「电商智能推荐系统」+ **「成熟客服智能体能力」**（话术风格/自动素材/销冠复制/场景策略/全自动发送）

---

## 一、项目定位

一个集 **商品管理 → 智能推荐 → 智能客服 → 下单转化 → 工单售后 → 数据运营** 于一体的全链路 AI 电商平台。

不是客服 + 推荐的简单拼凑，而是通过 **统一 Agent 编排层** 让推荐、客服、订单、营销四个子系统协同工作，形成数据闭环。

---

## 二、融合可行性分析

### 2.1 技术栈完全一致（合并零阻力）

| 层次 | 项目一·客服 | 项目八·推荐 | 融合后 |
|------|:----------:|:----------:|:------:|
| 前端 | Vue3 + Element Plus | Vue3 + 响应式 | ✅ 统一 |
| 后端 | FastAPI + SQLAlchemy | FastAPI + Celery | ✅ 统一 |
| AI | RAG + Agent | RAG + 协同过滤 | ✅ 互补 |
| 向量库 | ChromaDB | ChromaDB | ✅ 共享 |
| 缓存 | Redis | Redis | ✅ 共享 |
| 数据库 | MySQL | MySQL | ✅ 统一 |

### 2.2 功能互补形成闭环

```
用户浏览 ──→ 推荐引擎 ──→ 商品详情 ──→ AI客服咨询
    ↑                                       │
    │                                       ▼
用户画像 ←── 行为分析 ←── 订单完成 ←── 下单支付
    │                                       │
    └──── 售后工单 ←── 客服介入 ←── 退换诉求 ←┘
```

### 2.3 融合的核心价值

| 维度 | 融合前 | 融合后 |
|------|--------|--------|
| 商品知识库 | 两个系统各自维护 | 统一向量库，一处更新全平台同步 |
| 用户画像 | 客服和推荐互不知晓 | 浏览+购买+咨询+售后 全链路画像 |
| 意图识别 | 仅识别咨询/投诉 | 咨询→推荐引导→下单→售后 全意图覆盖 |
| 数据闭环 | 各自为政 | 客服高频问题→优化推荐；推荐失败→触发客服 |
| Agent 决策 | 单 Agent 独立决策 | 多 Agent 编排，主控路由子 Agent |

### 2.4 风险与应对

| 风险 | 应对策略 |
|------|---------|
| 模块膨胀到 8+ | 分三期交付，一期核心链路、二期营销、三期数据 |
| 数据库重构 | 从项目八 SPU/SKU 升级，兼容项目一简单结构 |
| Agent 冲突 | 引入 Master Agent 编排层，统一路由和上下文管理 |
| 前端整合 | 统一设计语言，按角色（消费者/管理员/客服）分视图 |

## 五、Agent 编排设计

### 5.1 Master Agent 路由策略

```
用户输入 ──→ Master Agent ──→ 意图分类
                │
     ┌──────────┼──────────┬──────────┐
     ▼          ▼          ▼          ▼
  咨询/投诉   浏览/推荐   下单/支付   营销/活动
     │          │          │          │
  客服Agent   推荐Agent   订单Agent   营销Agent
     │          │          │          │
  知识库      用户画像    订单系统    活动引擎
  + 工单      + 商品向量   + 支付      + 优惠券
```

### 5.2 Agent 间协同场景

| 场景 | 触发 | Agent 协作链 |
|------|------|-------------|
| 用户问"有什么好手机？" | 推荐Agent | 推荐 → 展示商品 → 附带"需要我帮你比价吗？" |
| 用户问"这个手机续航怎么样？" | 客服Agent | 客服(RAG查知识库) → 返回答案 → 附加"要加入购物车吗？" |
| 用户说"我要退货" | 订单Agent→客服Agent | 订单查订单 → 客服创建工单 → 引导填写退货原因 |
| 营销推送"618大促" | 营销Agent→推荐Agent | 营销匹配活动 → 推荐筛选活动商品 → 推送用户 |
| 客服发现某商品投诉激增 | 客服Agent→推荐Agent | 客服标记 → 推荐系统降权该商品 → 通知运营 |

### 5.3 共享上下文设计

```
Redis Session Store:
  session:{user_id}
    ├── chat_history: [...]     # 最近10轮对话
    ├── current_intent: str     # 当前意图
    ├── browsing_context: {     # 浏览上下文
    │     last_viewed: [...]
    │     cart_items: [...]
    │   }
    ├── user_profile: {...}     # 轻量画像摘要
    └── agent_state: str        # 当前活跃Agent
```

## 开发计划（三期交付）

| 阶段 | 周期 | 内容 |
|------|:---:|------|
| **一期：核心链路** | 3周 | 统一用户中心 + 商品中心(SPU/SKU) + 智能推荐引擎 + 购物车/订单 |
| **二期：AI赋能** | 3周 | Master Agent编排层 + 智能客服(RAG+意图) + 客服与推荐协同 |
| **三期：运营闭环** | 2周 | 工单/售后 + 营销活动 + 数据运营面板 + 数据闭环自动化 |

**总计：8周**
